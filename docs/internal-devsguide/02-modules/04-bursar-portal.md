# Module: Bursar Portal

## Purpose
The financial nerve center of the school. The Bursar uses this portal to set fee structures, generate invoices, and track payments (especially M-Pesa transactions).

## Public Interface
- **Access Route:** `/bursar/*`
- **Role Requirement:** `bursar`

## Implementation Notes
- **Fee Structures:** Fees are generated by linking an `amount` to an `academic_term` and optionally a specific `class_id`. If `class_id` is null, the fee applies globally to all students in the school.
- **Invoicing:** When invoices are generated, the system creates records in `public.invoices` and detailed line items in `public.invoice_items`.
- **M-Pesa Ledger:** The Bursar can view the status of `public.mpesa_stk_requests` to see which parents have initiated a payment.

## Dependencies
- Writes to: `public.fee_structures`, `public.invoices`, `public.invoice_items`, `public.fee_payments`, `public.salary_advances`
- Reads from: `public.mpesa_stk_requests`, `public.students`

## Known Limitations / Tech Debt
- The system currently only tracks M-Pesa STK push requests. C2B Paybill reconciliations (where a parent manually pays via the Safaricom SIM toolkit) are not yet fully implemented in the database schema.
